1. Contexto y objetivo

Avis Canarias, empresa del grupo Domingo Alonso, opera con el sistema de gestión de alquiler de vehículos Rentway, desarrollado por Jimpisoft. Este sistema centraliza reservas, contratos, tarifas y flota para todas sus oficinas en las Islas Canarias.

El contact center de Avis Canarias atiende consultas de clientes sobre precios de alquiler. Hasta ahora, para conocer el precio de un modelo concreto en unas fechas específicas, los agentes debían iniciar el proceso de reserva completo en Rentway — ocupando tiempo y generando ruido en el sistema.

Objetivo del proyecto
Desarrollar una herramienta web interna, de acceso rápido y sin instalación, que permita a cualquier agente del contact center consultar el precio de alquiler de un modelo de vehículo para unas fechas y oficina determinadas — en tiempo real y sin crear ninguna reserva.

¿Qué hace exactamente la herramienta?

  • Muestra el catálogo completo de 27 modelos de la flota Rentway Canarias
  • Permite seleccionar un modelo, unas fechas y una oficina
  • Consulta el precio en tiempo real directamente al sistema Rentway
  • Muestra precio por día y total con IGIC incluido
  • No genera ninguna reserva ni compromete disponibilidad
Confirmado por el API
El endpoint utilizado (getMultiplePrices) es una operación de solo lectura. No crea reservas, no bloquea disponibilidad, no modifica ningún dato en Rentway. Es exactamente la operación correcta para consultas de precio desde un contact center.

2. Arquitectura de la solución

3. El problema CORS y la red corporativa

¿Qué es CORS?

CORS (Cross-Origin Resource Sharing) es una política de seguridad de los navegadores. Cuando una página web en dominio-A.com intenta hacer una petición HTTP a dominio-B.com, el navegador la bloquea automáticamente a menos que el servidor de destino indique explícitamente que lo permite.

En nuestro caso: la web en marcocebrian55.github.io intentaba hacer un POST a ws-soap-api.jimpisoft.com. El servidor de Jimpisoft no incluye las cabeceras CORS necesarias, por lo que el navegador bloqueaba todas las peticiones.

Error en consola del navegador
Access to fetch at 'https://ws-soap-api.jimpisoft.com/...' from origin 'https://marcocebrian55.github.io' has been blocked by CORS policy: No 'Access-Control-Allow-Origin' header is present on the requested resource.

Solución: Cloudflare Worker como proxy

Se creó un Cloudflare Worker — un script que corre en los servidores de Cloudflare, no en el navegador — que actúa como intermediario. El navegador hace la petición al Worker (que sí permite CORS), y el Worker reenvía la petición a Jimpisoft y devuelve la respuesta con las cabeceras correctas.

Complicación: Firewall corporativo de Domingo Alonso
La red corporativa de Domingo Alonso aplica una política de inspección SSL (Deep Packet Inspection) que bloqueaba tanto las peticiones directas a Jimpisoft como las peticiones a Cloudflare Workers. La solución para uso en oficina: hotspot de datos móviles o acceso desde red externa (casa, etc.). Fuera de la red corporativa, el proxy Cloudflare funciona directamente sin ninguna extensión.

Código del Cloudflare Worker

addEventListener('fetch', event => {
  event.respondWith(handleRequest(event.request));
});

async function handleRequest(request) {
  const corsHeaders = {
    'Access-Control-Allow-Origin': '*',
    'Access-Control-Allow-Methods': 'POST, OPTIONS',
    'Access-Control-Allow-Headers': 'Content-Type, SOAPAction',
  };

  if (request.method === 'OPTIONS')
    return new Response(null, { headers: corsHeaders });

  const url = 'https://ws-soap-api.jimpisoft.com/getMultiplePrices.asmx';
  const body = await request.text();

  const upstream = await fetch(url, {
    method: 'POST',
    headers: {
      'Content-Type': 'text/xml; charset=utf-8',
      'SOAPAction': '"http://www.jimpisoft.pt/Rentway_Reservations_WS/getMultiplePrices/MultiplePrices"',
    },
    body: body,
  });

  const text = await upstream.text();
  return new Response(text, {
    status: upstream.status,
    headers: {
      'Content-Type': 'application/xml; charset=utf-8',
      'Access-Control-Allow-Origin': '*',
    },
  });
}

4. Análisis del API SOAP de Rentway

Rentway expone sus servicios a través de un API SOAP 1.2 (Simple Object Access Protocol). A diferencia de los APIs REST modernos (que usan JSON), SOAP usa XML estructurado según unas reglas muy estrictas. Cualquier error en la estructura del XML — un campo mal nombrado, un namespace incorrecto — provoca error en el servidor sin dar información útil.

Descubrimiento del contrato: el WSDL

Los APIs SOAP publican un documento WSDL (Web Services Description Language) que describe exactamente cómo estructurar las peticiones: qué namespace usar, qué se llama cada operación, qué campos son obligatorios, etc.

URL del WSDL de Jimpisoft: https://ws-soap-api.jimpisoft.com/getMultiplePrices.asmx?WSDL

Estructura de una petición SOAP correcta
<?xml version="1.0" encoding="utf-8"?>
<soap:Envelope
  xmlns:soap="http://www.w3.org/2003/05/soap-envelope"
  xmlns:tns="http://www.jimpisoft.pt/Rentway_Reservations_WS/getMultiplePrices">
  <soap:Header/>
  <soap:Body>
    <tns:MultiplePrices>
      <tns:objRequest>
        <tns:companyCode>200037</tns:companyCode>
        <tns:customerCode>18</tns:customerCode>
        <tns:username>Booking</tns:username>
        <tns:password>···</tns:password>
        <tns:groupID>SC</tns:groupID>
        <tns:rateCode>AG1LPA</tns:rateCode>
        <tns:pickUp>
          <tns:Date>2026-06-10T10:00:00</tns:Date>
          <tns:rentalStation>LPA</tns:rentalStation>
        </tns:pickUp>
        <tns:dropOff>
          <tns:Date>2026-06-14T10:00:00</tns:Date>
          <tns:rentalStation>LPA</tns:rentalStation>
        </tns:dropOff>
      </tns:objRequest>
    </tns:MultiplePrices>
  </soap:Body>
</soap:Envelope>

Formato de la respuesta

Jimpisoft responde con un XML de tipo ADO.NET DataSet diffgram, formato propio de aplicaciones .NET. Los datos reales están dentro del bloque <diffgr:diffgram> dentro del elemento <MultiplePrices>. Los campos de precio relevantes son:

Campo XMLSignificado
totalDayValueWithTaxPrecio por día con IGIC incluido
previewValuePrecio total del alquiler (todos los días)
nrDaysNúmero de días calculados por el sistema
rateCodeCódigo de tarifa aplicado
stationIDID de la oficina
group_NameNombre del grupo/categoría del vehículo
taxRatePorcentaje de IGIC (15% en Canarias)
errorCodeTexto del error si la petición falla

5. Los 5 bugs identificados y cómo se resolvieron

1

Double response.text() — el bug del precio 0€

El código llamaba a response.text() dos veces sobre el mismo objeto de respuesta HTTP. En JavaScript, el stream de respuesta solo puede leerse una vez. La segunda llamada devuelve siempre una cadena vacía, por lo que el parser XML recibía un documento vacío y los precios eran siempre 0€.

Antes (incorrecto)Después (correcto)
const responseText = await response.text();
// stream ya consumido ↑
const textData = await response.text();
// textData siempre es ""
const xml = parser.parseFromString(textData, ...);
 
const responseText = await response.text();
// reusar la misma variable
const xml = parser.parseFromString(responseText, ...);
2

URL directa a Jimpisoft sin proxy

La URL apuntaba directamente al servidor de Jimpisoft, lo que causaba el error CORS inmediato. Se redirigió al Cloudflare Worker y se corrigió el Content-Type.

El campo de la estación de recogida se llamaba Rental_Station (con mayúscula y guión bajo) en el código, pero en el WSDL oficial se llama rentalStation (camelCase). Este error causaba el mensaje "ERROR: PICKUP STATION DOES NOT EXIST!" para cualquier oficina que se introdujera.

¿Por qué solo funcionan las tarifas AG1?
Rentway gestiona el acceso a tarifas por canal (customerCode). El usuario Booking / customerCode 18 corresponde al canal de booking web de Avis, que tiene asignadas las tarifas AG1 por aeropuerto. Las tarifas corporativas (ACD, ACN), de agencia (AGS4, AGNS4) y "Avis Directo" (ADTxxx) pertenecen a otros canales con sus propias credenciales. No es un error — es el diseño de permisos de Rentway.

7. Resultado final y funcionamiento

La herramienta está publicada en https://marcocebrian55.github.io/FLOTA_DAG/ y funciona completamente. El flujo de uso para una agente del contact center es:

1

Abrir la web e identificar el modelo

El catálogo muestra los 27 modelos de la flota con imagen, categoría, grupo y código SIPP. La agente puede filtrar por categoría o buscar por nombre/marca.

2

Seleccionar el modelo y abrir el calculador

Un clic en la tarjeta lo selecciona (tick azul). Con al menos un modelo seleccionado, se activa el botón "Calcular precio/día" en la barra superior.

3

Rellenar el formulario

Se abre un modal con 3 campos: fecha de recogida, fecha de devolución y oficina. La tarifa se auto-rellena automáticamente según la oficina elegida (AG1 + código de oficina).

4

Obtener el precio en tiempo real

Al hacer clic en "Solicitar Precios a Rentway", la herramienta envía la consulta y muestra en segundos: precio por día (con IGIC), precio total, número de días y tarifa aplicada.

Resultado confirmado en pruebas reales
Volkswagen Polo · Gran Canaria Aeropuerto · 10–14 junio 2026 · Tarifa AG1LPA
→ 22,50 € / día · 90,00 € total (4 días, IGIC 15% incluido)

8. Preguntas clave sobre el sistema

¿Podría cada proveedor introducir su usuario y contraseña para ver sus tarifas propias?

Sí, es totalmente posible. El API SOAP de Jimpisoft/Rentway acepta en cada petición los campos companyCode, customerCode, username y password. Esto significa que la herramienta puede ampliarse para que cada empresa del grupo (Avis, Budget, Cicar, u otras que operen en Rentway) tenga acceso a sus propias tarifas introduciendo sus credenciales específicas.

Ejemplo práctico
Si Budget tiene en Rentway un companyCode diferente y sus propias credenciales, podría ver sus tarifas de Budget (con sus grupos de vehículos propios, sus oficinas y sus precios específicos) sin que interfieran con las de Avis. Bastaría con añadir un selector de empresa en el formulario y mapear cada empresa a sus credenciales correspondientes.

Lo que necesitaríamos para implementarlo:

  • Obtener el companyCode, customerCode, username y password de Rentway para cada empresa
  • Añadir un selector de empresa en la interfaz
  • Adaptar el catálogo de flota a cada empresa (modelos, grupos, imágenes)
  • El proxy Cloudflare Worker y toda la lógica SOAP no necesitarían modificación
Conclusión
Sí, es posible y no sería complejo de implementar. La arquitectura actual ya soporta multi-proveedor — solo habría que añadir la lógica de selección de empresa y sus credenciales correspondientes.

¿Esto resuelve el problema de conocer el precio sin tener que crear una reserva?

Sí, y de hecho este es el objetivo principal para el que está diseñado el endpoint getMultiplePrices. Se trata de una operación de solo lectura en el sistema Rentway:

Crear una reserva normalgetMultiplePrices (nuestra herramienta)
Genera un registro en RentwayNo crea ningún registro
Bloquea disponibilidad temporalmenteNo toca la disponibilidad
Requiere datos del cliente (nombre, DNI…)Solo necesita modelo, fechas y oficina
Hay que cancelarla si el cliente no confirmaNo hay nada que cancelar
Proceso de 5–10 minutos para el agenteRespuesta en 2–3 segundos
Confirmado en pruebas
Durante el desarrollo se realizaron decenas de consultas reales al sistema Rentway. Ninguna generó reservas ni registros en el back-office. La herramienta consulta y devuelve precios reales del sistema, pero sin ningún efecto secundario.

Para el contact center esto supone un cambio significativo: en lugar de iniciar un proceso de reserva completo para informar de un precio al cliente (y luego cancelarlo si no confirma), la agente puede obtener el precio exacto en segundos, informar al cliente, y si confirma, entonces sí abrir el proceso de reserva real en Rentway.

9. Escalabilidad: multi-proveedor y futuro

¿Puede cada proveedor usar sus propias credenciales?

Sí. El API SOAP de Jimpisoft acepta companyCode, customerCode, username y password en cada petición. Si otros proveedores del grupo (Budget, Cicar, u otros que operen en Rentway) tienen sus propias credenciales, la herramienta puede ampliarse con un selector de empresa. Cada empresa vería únicamente sus tarifas, modelos y oficinas propias.

¿Qué requeriría esta ampliación?

  • Obtener los companyCode y credenciales de cada empresa en Rentway
  • Añadir un selector "Empresa" al formulario
  • Cargar el catálogo de flota correspondiente a cada empresa
  • El código SOAP y el proxy Cloudflare Worker no necesitarían modificación
